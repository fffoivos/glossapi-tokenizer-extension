Important integration note: do not call the package's high-level
`TokenDistillation.run(...)` entry point for Apertus production. It appends
tokens with `add_tokens(...)`; our ReTok arm uses an exact merge-extended
tokenizer with fixed IDs. The Apertus adapter should call the lower-level
training loop with an explicit `[131072, 148480)` mapping.
