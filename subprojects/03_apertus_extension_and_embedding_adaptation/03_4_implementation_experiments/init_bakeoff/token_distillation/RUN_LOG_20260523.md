## TD wrapper hardening

Updated `train_retok_td.py` and `train_retok_td.sbatch` so a selected token is
trained only if enough snippets reproduce its static base phrase under the base
tokenizer. Tokens that fail this check are recorded in
`retok_td_manifest.json` under `skipped_tokens` and remain frozen/preserved at
their existing ReTok initialization.

Default smoke behavior:

- `MIN_ACCEPTED_SNIPPETS_PER_TOKEN=$SNIPPETS_PER_TOKEN`
- `MIN_TRAINED_TOKEN_FRACTION=0.90`

This means a smoke can tolerate a small number of unstable/noisy selected
tokens, but fails before model load if too much of the selection is untrainable.

Validation:
