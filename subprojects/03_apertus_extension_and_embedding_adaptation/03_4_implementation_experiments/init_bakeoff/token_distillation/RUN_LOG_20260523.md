## TD wrapper hardening

Updated `train_retok_td.py` and `train_retok_td.sbatch` so a selected token is
trained only if enough snippets reproduce its static base phrase under the base
tokenizer. Tokens that fail this check are recorded in
`retok_td_manifest.json` under `skipped_tokens` and remain frozen/preserved at
their existing ReTok initialization.

Default smoke behavior:

- `MIN_ACCEPTED_SNIPPETS_PER_TOKEN=$SNIPPETS_PER_TOKEN`
- `MIN_TRAINED_TOKEN_FRACTION=0.90`

This means a smoke can tolerate a small number of unstable/noisy selected
tokens, but fails before model load if too much of the selection is untrainable.

Validation:

- Local: `python3 -m py_compile train_retok_td.py`
- Clariden with job runtime env:
  `uenv run pytorch/v2.9.1:v2 --view=default -- python3 -m py_compile train_retok_td.py`

## Preservation verifier hardening

Added and then hardened the post-TD preservation verifier:

- Commit `6b2473c` - add `verify_td_preservation.py` and
  `verify_td_preservation_xfer.sbatch`.
- Commit `bec2d02` - remove the torch dependency by parsing Safetensors
  directly, so the check can run on `xfer` as CPU-only work.
- Commit `993a82d` - make the verifier compatible with `xfer`'s Python version
  by replacing `int | None` annotations with `Optional[int]`.

The verifier now checks, from checkpoint artifacts only:

- state-dict key equality;
- tensor shape and dtype equality;
- exact byte preservation for every non-embedding tensor;
- exact byte preservation for all input/output embedding rows except the
  trained new-token IDs;
- every trained new input/output row changed;
- trained rows are not all-zero and have no BF16/F16/F32 inf/nan bit patterns;
- xIELU and QK-Norm tensors are covered by the non-embedding exact check.
